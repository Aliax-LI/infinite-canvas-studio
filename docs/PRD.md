## 2. 产品愿景

Infinite Canvas Studio 是一款本地优先的 AI 无限创作画布。它把项目、画布、素材、工作流、模型供应商和生成任务放进一个中文桌面工作区，让用户在没有自有账号、没有网络、没有 API Key 时仍能整理素材与编辑画布；只有用户主动调用在线服务或检查更新时才联网。

`1.0.0` 的完成标准不是界面像原版，而是冻结基线中的用户可见核心能力均有对应实现和验收记录。允许重新设计界面、数据模型与画布引擎，不允许静默遗漏功能。

## 3. 目标与非目标

### 3.1 目标

- 完整复刻冻结基线的核心能力，并用功能矩阵双向追踪。
- 使用 Electron、React、shadcn/ui Nova、Tailwind CSS v4 重建桌面体验。
- 使用模块化 Python/FastAPI sidecar 承载业务，不延续巨型单文件后端。
- 使用本机 SQLite 保存结构化数据，使用本地盘、外置盘或 NAS/SMB 保存大型文件。
- 同时交付 Windows x64 与 macOS Apple Silicon 安装包。
- 同时交付 Chrome 采集扩展和 Photoshop UXP 连接器。
- 默认无遥测；诊断信息只在本地生成并由用户主动分享。

### 3.2 非目标

- 不兼容原版 JSON 数据目录，也不提供旧数据迁移器。
- `1.0.0` 不提供英文界面、Linux、Intel Mac、触控屏或数位笔正式支持。
- 不提供多人实时协作、自有账号系统或项目云同步。
- OneDrive、iCloud Drive、Dropbox 等同步目录不属于正式存储场景。
- 不承诺未签名 macOS 应用的通知中心提醒或自动覆盖升级。
- 不保留基线中的“在线人数”显示；实时事件通道仍保留。

## 4. 用户与核心场景

### 4.1 目标用户

- 使用多个文本、图片、视频模型的个人创作者。
- 使用 ComfyUI、RunningHub 或自定义 OpenAI 兼容服务的高级用户。
- 需要从网页、Photoshop 和本地目录集中采集素材的视觉工作者。

### 4.2 核心旅程

1. 首次启动确认资料库存储位置，可跳过模型和配套组件配置。
2. 新建或打开项目，在单主窗口的多画布标签中组织节点与连线。
3. 从磁盘、共享文件夹、Chrome 或 Photoshop 导入素材。
4. 配置模型供应商或 ComfyUI，提交生成任务并在任务中心追踪。
5. 编辑、比较、复用生成结果，使用历史、撤销和崩溃恢复保护工作。
6. 导出画布或项目，在另一台电脑通过公开格式导入。
7. 检测到新版本后查看说明并打开对应 GitHub Release 页面手动安装。

## 5. 功能需求

### 5.1 桌面外壳与首次启动

- 首次启动必须显示推荐资料库路径、剩余空间并要求用户确认。
- 资料库缺失时进入恢复页，不自动创建空库。
- 默认恢复上次项目、画布、视口与面板状态；异常退出时显示恢复入口。
- 单主窗口支持多画布标签；媒体可在轻量独立窗口预览。
- Windows 无任务时关闭即退出；macOS 遵循关闭窗口但应用仍运行的惯例。
- 有活动任务时后台继续运行并提供任务中心、托盘或 Dock 提醒。
- 支持开机启动，但默认关闭；同一资料库禁止多实例写入。

### 5.2 画布与编辑

- 使用新的 React 画布引擎实现节点、端口、连线、平移缩放、多选、框选、分组、复制粘贴、层级、对齐和快捷操作。
- 快捷键固定并按 Windows/macOS 映射；命令面板可检索所有主要操作。
- 每张画布默认保留最近 100 次跨重启撤销/重做记录。
- 剪贴板内容安全标准化：文本、图片、文件和 URL 分别转为受控节点，不保留外部 HTML 样式。
- 图片裁剪、遮罩、画笔、比较和全景等专用能力可使用 Canvas 2D/Three.js。
- 常规界面满足 WCAG 2.2 AA 基础要求；画布提供键盘操作与节点列表辅助视图。

### 5.3 项目、素材与文件

- 单一本机 SQLite 保存项目、画布、节点、连线、历史、会话、任务和索引。
- 大型文件以 SHA-256 内容寻址，自动去重并保留原文件名、来源和标签。
- 普通导入和共享文件夹导入都复制到受管理存储；源文件变化只可作为新版本导入。
- 受管理文件可位于本机、外置盘或共享 NAS/SMB；SQLite 始终位于本机。
- 多台设备可共享同一 NAS 文件池，但不共享项目和数据库；NAS 文件不可自动物理删除。
- 删除内容进入 7 天回收站；成功生成结果永久保留，缓存和失败中间文件可清理。
- 素材缺失时只扫描资料库、登记目录或用户指定目录，不扫描整块硬盘。

### 5.4 AI、工作流与任务

- 支持内置适配器和自定义供应商；按文本、图片、视频能力设置默认模型。
- 覆盖冻结基线的 OpenAI 兼容、Gemini、火山、RunningHub、ModelScope、ComfyUI、CLI 和异步任务协议。
- 多个 ComfyUI 实例由调度器按在线状态、队列和能力自动选择。
- 任务队列持久化，默认全局并发 4、单在线供应商 2、单 ComfyUI 实例 1，并允许设置。
- 任务失败立即报错，不自动重试；用户可查看错误并主动再次执行。
- 历史保存提示词、模型、参数、种子、状态和结果引用，不保存密钥、请求头、Base64 或完整原始响应。

### 5.5 配套组件

- Chrome Manifest V3 扩展和 Photoshop 24.0+ UXP 插件属于 `1.0.0` 范围。
- 首次连接必须通过一次性配对；两个组件使用独立、可撤销令牌。
- 协议主版本不兼容时阻止连接，次版本差异只提醒。
- Chrome 扩展拥有全站权限，但只在用户主动采集时读取当前页面，不记录浏览历史或后台扫描。
- Chrome 采集默认插入当前画布；无活动画布时进入采集收件箱。
- Chrome ZIP 与 Photoshop `.ccx` 均通过 GitHub Release 分发，首次引导按需提示安装。

### 5.6 备份、恢复与导入导出

- 每日自动备份 SQLite、工作流和轻量配置，默认保留 3 份；升级前另建保护备份。
- 完整媒体备份由用户手动触发，并校验文件完整性。
- 整库恢复前先保护当前状态；项目与画布包可合并导入。
- 崩溃恢复版本不得覆盖正式画布，用户可预览后恢复、另存或丢弃。
- `.icproject` 与 `.icanvas` 为公开、版本化 ZIP 格式，支持双击预览后导入。
- 任何导出、备份或诊断包都不得包含 API Key 和配对令牌。
- 数据库不支持降级写入；旧版本遇到新架构时必须拒绝打开。

### 5.7 设置、安全与隐私

- API Key 可按用户决定明文保存在本机独立配置表；文件权限受限，日志和导出必须脱敏。
- Python 内部接口只监听 `127.0.0.1`，每次启动使用临时会话令牌。
- React 通过 HTTP/SSE/WebSocket 使用业务接口；文件选择、窗口、托盘和外链通过最小 Electron IPC。
- Electron 启用 Context Isolation、Sandbox、严格 CSP，禁用 Node Integration，不嵌入远程网页。
- 外部链接只允许 HTTPS，并交由系统浏览器打开。
- 一键诊断包不包含提示词正文、画布内容、素材或密钥，并在导出前展示清单。

### 5.8 更新、发布与卸载

- 应用只检查 GitHub Release；发现新版本后打开当前平台的安装包页面，由用户手动安装。
- 发布通道分为开发版、Beta 和稳定版；默认只提示稳定版。
- Windows 发布 NSIS EXE 和 ZIP；macOS 发布 arm64 DMG 和 ZIP；所有产物附 SHA-256。
- Windows 卸载程序可选择删除全部数据，默认不勾选并二次确认。
- macOS 提供“卸载与清除数据”助手，再提示用户移除应用本体。
- 安装包内置按平台构建的 Python sidecar 和 LGPL FFmpeg/ffprobe，并提供许可证与构建说明。

## 6. 非功能要求

| 维度     | `1.0.0` 门槛                                                                    |
| -------- | ------------------------------------------------------------------------------- |
| 性能     | 16GB 设备上支持 1,000 节点、2,000 连线、50,000 素材索引；常规搜索通常小于 300ms |
| 可靠性   | 不得存在数据丢失、安装/启动失败、高频崩溃或核心功能缺失                         |
| 安全     | 仅本机监听、令牌鉴权、CSP、IPC 白名单、日志脱敏、路径校验                       |
| 可访问性 | 常规 UI 达到 WCAG 2.2 AA 基础要求，支持键盘和减少动态效果                       |
| 可测试性 | 每个功能矩阵条目必须关联自动化测试或人工验收记录                                |
| 包体积   | 单平台下载约 180–300MB，安装后约 400–700MB，以可靠性优先                        |

## 7. 发布门槛

`1.0.0` 只有在以下条件全部满足时才能发布：

- 功能复刻矩阵无未批准的缺失项。
- Windows x64 与 macOS Apple Silicon 安装、启动、保存、恢复、卸载冒烟通过。
- Chrome 与 Photoshop 端到端配对及核心流程通过。
- CI 契约测试通过，并使用用户提供的真实账号完成外部服务冒烟。
- 无已知数据丢失、安全、核心功能、高频崩溃或安装启动阻断问题。
- 许可证、原作者署名、第三方许可证和 FFmpeg 源码说明齐全。

低影响视觉或边缘缺陷可以作为已知问题发布，但必须在 Release Notes 中明确记录。
