# IC-SHELL-002：资料库首次引导

- 矩阵编号：`IC-SHELL-002`
- 上游锚点：无直接上游实现；这是 `docs/PRD.md` 5.1 和
  `docs/TECHNICAL_ARCHITECTURE.md` 4.1 规定的目标架构行为。
- 目标模块：`apps/desktop`、`packages/contracts`、`apps/renderer`。

## 已实现

- Electron 先读取位于应用用户目录的最小引导配置，再决定是否启动 sidecar；
  引导配置仅保存版本和资料库根路径，不保存数据库内容或凭证。
- 首次启动显示推荐资料库路径和该文件系统的可用空间。用户可接受推荐路径，或通过
  系统目录选择器改选位置；必须再次点击“确认并继续”后才创建目录、保存配置和启动服务。
- 已配置路径不可访问时进入恢复引导，展示原路径并要求重新选择；不会自动创建空资料库。
- 资料库根目录始终作为 `ICS_LIBRARY_ROOT` 传给 Python sidecar，渲染器只经受限 IPC
  获取引导状态、目录选择、空间检查和配置请求。

## 当前证据

- 桌面单元测试覆盖配置持久化、可用空间读取和已保存路径缺失时的恢复状态，且确认不会
  重建缺失目录。
- Windows Electron 运行时人工验证：首次打开显示推荐路径
  `Documents/Infinite Canvas Studio` 与可用空间；选择“使用推荐位置”后，确认按钮才可用；
  确认后进入已启动 sidecar 的项目工作台。

## 未完成验收

`IC-SHELL-002` 仍处于“实现中”。目录选择器的 macOS 人工验证、路径不可访问恢复页的
Electron 端到端脚本，以及引导后的模型/配套组件可跳过流程尚未完成。
