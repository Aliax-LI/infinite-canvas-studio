# 分阶段开发任务

## 执行规则

- 以可安装、可启动、可保存的纵向切片推进；Alpha 可以不完整，`1.0.0` 不可以。
- 每个任务必须关联一个或多个 `IC-*` 条目，并补充自动测试或人工证据。
- 每阶段结束都在 Windows x64 与 macOS arm64 产出可运行包。
- 任何数据丢失、安全、安装启动或核心功能回归都阻止进入下一阶段。

## P0：仓库与冻结基线

目标：建立可持续开发和功能追踪的可信起点。

- [x] 建立 pnpm monorepo、Electron、React/shadcn、Python sidecar 与扩展骨架。
- [x] 固定产品需求、技术架构、功能矩阵和阶段任务。
- [x] 以 submodule 固定 `hero8152/Infinite-Canvas@bc7efbd`，标记为只读参考。
- [x] 建立 Windows/macOS CI、Release 构建、SHA-256 与制品保留。
- [x] 建立第三方许可证清单和上游来源追踪规则。
- [ ] 完成基线二次审计，把所有页面、节点、路由、快捷键和插件命令映射到矩阵。

退出门槛：全新克隆可安装依赖，格式、lint、类型、单测、构建和矩阵检查全部通过；双端 CI 至少完成一次。

## P1：桌面纵向切片

目标：打通“Electron → sidecar → SQLite → React”的最小真实数据流。

- [ ] 实现 sidecar supervisor：随机端口、临时令牌、健康检查、限次重启和优雅退出。
- [ ] 建立受限 preload API 与 IPC 参数校验。
- [ ] 建立 FastAPI 鉴权、中间件、统一错误、SSE/WebSocket 基础设施。
- [ ] 引入 SQLAlchemy、Alembic 和首个 schema；只允许 Python 访问数据库。
- [ ] 首次启动确认资料库、缺失恢复页、单实例锁和安全退出。
- [ ] CI 打包 PyInstaller sidecar 并随 Electron 安装包运行。

退出门槛：双端安装后可选择资料库、创建项目、保存并重启恢复；断开 sidecar 能显示可操作诊断。

## P2：画布与项目核心

目标：交付可承担真实工作量的无限画布。

- [ ] 引入 `@xyflow/react` 并实现画布文档/命令边界。
- [ ] 实现节点、端口、连线、多选、框选、分组、层级、对齐和命令面板。
- [ ] 实现项目、画布、多标签、视口与面板状态持久化。
- [ ] 实现操作日志、100 次撤销/重做、快照和崩溃恢复候选版本。
- [ ] 实现文本、图片、视频和基础工作流节点壳。
- [ ] 建立 1,000 节点/2,000 连线性能基准与虚拟化策略。
- [ ] 完成键盘操作、节点列表辅助视图和常规 UI 无障碍基线。

退出门槛：核心画布矩阵通过；自动保存不阻塞；强制杀进程后可选择恢复。

## P3：素材与存储

目标：安全管理大量本地与共享媒体。

- [ ] 建立文件存储端口与本地、外置盘、NAS/SMB 驱动。
- [ ] 实现 SHA-256 内容寻址、原子写入、去重、来源和引用计数。
- [ ] 实现普通导入、共享文件夹浏览、缩略图、搜索、标签和预览。
- [ ] 集成 LGPL FFmpeg/ffprobe 的双平台探测、首帧和抽帧。
- [ ] 实现 7 天回收站、缓存清理、存储分析和素材丢失修复。
- [ ] 实现共享 NAS 引用清单和只允许手动执行的全局清理。
- [ ] 实现“复制—校验—切换”的存储迁移。

退出门槛：50,000 素材索引性能达标；拔出外置盘或断开 NAS 不损坏 SQLite，且恢复路径明确。

## P4：供应商、工作流与任务

目标：以统一契约复刻所有 AI 调用能力。

- [ ] 建立供应商注册、能力标签、默认模型和本机凭证配置。
- [ ] 实现持久化任务状态机、并发限制、取消、进度和用户重试。
- [ ] 逐个迁移 OpenAI 兼容、Gemini、火山、RunningHub、ModelScope 和 CLI 适配器。
- [ ] 实现多 ComfyUI 实例探测、能力发现、工作流导入和自动调度。
- [ ] 实现生成历史、可复现参数和敏感数据排除。
- [ ] 为每个协议建立模拟契约测试；准备真实账号冒烟脚本。

退出门槛：所有供应商/ComfyUI P0 矩阵条目通过契约测试；断网、限流、超时和不明确提交状态均不重复计费。

## P5：图片、视频与高级节点

目标：补齐冻结基线中的专用创作能力。

- [ ] 完成图片裁剪、遮罩、画笔、比较与导出。
- [ ] 完成视频预览、信息、首帧、抽帧与结果节点。
- [ ] 完成全景/Three.js 等基线预览能力。
- [ ] 审计并实现剩余节点类型、参数面板、输入输出映射和异常状态。
- [ ] 验证 GPU 默认启用和软件渲染安全模式。

退出门槛：节点与编辑领域矩阵无未审计 P0 条目；Windows/macOS 输出一致。

## P6：Chrome 与 Photoshop

目标：交付可由普通用户安装的配套组件。

- [ ] 实现固定本机集成网关、一次性配对、独立令牌和撤销。
- [ ] 实现协议主/次版本兼容检查和权限能力表。
- [ ] 完成 Chrome 主动采集、右键/选区、当前画布插入和采集收件箱。
- [ ] 完成 Photoshop 24.0+ 置入、导出当前文档、刷新与生成调用。
- [ ] 打包 Chrome ZIP 和 Photoshop CCX，完善首次启动与设置引导。

退出门槛：新用户按 Release 文档可完成安装、配对、采集/交换和撤销；网页脚本无法取得令牌。

## P7：备份、交换、更新与运维

目标：让数据可恢复、可迁移、可诊断。

- [ ] 实现 3 份每日自动备份、升级保护备份和完整媒体备份。
- [ ] 实现整库恢复、引用检查和恢复前保护。
- [ ] 发布 `.icproject`/`.icanvas` 开放格式规范、校验和安全解压。
- [ ] 实现文件关联、预览后导入和跨设备项目迁移。
- [ ] 实现稳定/Beta Release 检查和平台下载页跳转。
- [ ] 实现脱敏诊断包、日志轮换和用户预览。
- [ ] 实现 Windows 卸载数据选项与 macOS 清除数据助手。

退出门槛：备份/恢复故障注入通过；恶意导出包不能路径穿越；所有导出均不含密钥或配对令牌。

## P8：`1.0.0` 发布候选

目标：用证据关闭全部发布门槛。

- [ ] 功能矩阵逐项审计，所有 P0/P1 为“已验收”或“批准差异”。
- [ ] 在最低支持系统完成安装、启动、升级提示、卸载和数据保留矩阵。
- [ ] 使用真实账号完成所有承诺供应商冒烟，记录供应商侧版本和日期。
- [ ] 完成安全审查：CSP、IPC、回环接口、配对、凭证、路径和诊断脱敏。
- [ ] 完成性能、长时间任务、异常断电、磁盘满、外置盘/NAS 断线测试。
- [ ] 完成许可证、上游署名、FFmpeg 源码/构建说明和第三方通知。
- [ ] 发布 `1.0.0-beta.n`，关闭阻断问题后发布 `1.0.0`。

最终门槛：不存在核心功能缺失、数据丢失、安全、安装启动或高频崩溃问题；仅允许已记录的低影响已知问题。
